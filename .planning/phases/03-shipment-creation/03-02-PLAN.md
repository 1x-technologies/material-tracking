  <action>
    1. Create `apps/api/src/routers/shipment.ts` with **`staffProcedure`** for all mutations/queries in this task.

    2. **`create`** — input `createShipmentSchema`:
       - Run Firestore **transaction**:
         - Read `counters/shipments` — if missing, treat `last` as 0; increment to next int; store back.
         - Generate `shipmentNumber` string: **`SH-{UTC_YYYYMMDD}-{seq}`** where `seq` is zero-padded 4 digits (e.g. `SH-20260401-0001`).
         - Resolve `originId` and `destinationId` against `locations` docs; if either missing or inactive, throw `BAD_REQUEST` with message containing **`INVALID_LOCATION`**.
         - Build `origin` / `destination` as `LocationRef`: `{ locationId: doc.id, name: data.name }` (use `name` field from Location).
         - Create `shipments/{autoId}` with: `shipmentNumber`, `status: "created"`, `priority`, `category`, `description`, `pieceCount`, `sender`, `receiver`, `origin`, `destination`, `createdBy: { uid: ctx.user.uid, name: ctx.user.name ?? ctx.user.email ?? "" }`, `createdAt`, `updatedAt` server timestamps, `notificationPrefs` from input or defaults `{ onTransit: false, onDelivery: true, onPickup: true }`.
         - Batch create subcollection **`pieces`** documents: for `i` in `1..pieceCount`, doc id auto-generated, fields: `shipmentId`, `pieceNumber: i`, `qrCode: "pending:"+autoId+":"+i`, `status: "created"`, `events: []`, `currentLocation: origin`, `photoUrls: []`, `updatedAt` server timestamp (add other `Piece` fields as required by type with safe defaults).
       - Return `{ shipmentId, shipmentNumber }`.

    3. **`getById`** — input `z.object({ shipmentId: z.string() })` — return shipment doc data + id; if missing `NOT_FOUND`.

    4. **`update`** — input `z.object({ shipmentId: z.string(), patch: updateShipmentInputSchema })` — load shipment; if `status !== "created"`, throw `CONFLICT` with message **`SHIPMENT_NOT_EDITABLE`**; merge allowed fields; `updatedAt` server timestamp; if `pieceCount` changes, **document in code comment** that piece rows are not auto-adjusted in Phase 3 (executor: either reject pieceCount change in update **or** no-op pieceCount in update schema — **reject** `pieceCount` in update for Phase 3: remove from `updateShipmentInputSchema` usage if present, or strip in procedure) — **Concrete:** do **not** allow `pieceCount` in update mutation input; only allow description, category, priority, sender, receiver, originId, destinationId, notificationPrefs. If schema still has pieceCount optional, **omit** from merge in procedure.

    5. **`cancel`** — input `cancelShipmentInputSchema` — if status not `created`, throw `CONFLICT` **`SHIPMENT_NOT_CANCELLABLE`**; else set `status: "cancelled"`, `updatedAt` server timestamp.

    6. Mount `shipment: shipmentRouter` in `apps/api/src/router.ts`.
  </action>
