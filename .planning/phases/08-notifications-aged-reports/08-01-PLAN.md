   - Import `sendNotificationEmail` from `../lib/email`
   - Import template builders from `../lib/emailTemplates`

   - Export `onShipmentStatusChange` using `onDocumentUpdated` with:
     - `document: "shipments/{shipmentId}"`
     - `region: "us-central1"`

   - Handler logic:
     a. Extract `before` and `after` data from `event.data`
     b. Early return if no data or `before.status === after.status` (no status change)
     c. Extract `newStatus = after.status` and `shipmentId = event.params.shipmentId`
     d. Build `ShipmentEmailData` from `after` data: `shipmentNumber`, `status: newStatus`, `pieceCount`, `senderName: after.sender.name`, `receiverName: after.receiver.name`, `detailUrl` using shipmentId

   - **Notification decision logic** (per D-04):
     - If `newStatus === "delivered"` AND `after.notificationPrefs.onDelivery` → build delivered email
     - If `newStatus === "picked_up"` AND `after.notificationPrefs.onPickup` → build picked up email
     - If `newStatus === "in_transit"` AND `after.notificationPrefs.onTransit` → build in-transit email (NOTF-03 opt-in)
     - For `partially_delivered`, `created`, `cancelled` → no notification, return early

   - **Recipient resolution** (per D-05):
     - Collect recipients: always include `after.sender.email` (sender always has email per ShipmentSender type)
     - Include `after.receiver.email` only if it exists (optional on ShipmentReceiver — external receivers may not have email)
     - Filter out empty/undefined emails
     - Send one email per valid recipient using `sendNotificationEmail`
     - Log: `"Sending ${newStatus} notification for ${after.shipmentNumber} to ${recipients.length} recipients"`

2. **Update `apps/functions/src/index.ts`**:
