  <files>apps/web/src/pages/ScanPage.tsx</files>
  <read_first>

  <action>
    **Modify `apps/web/src/pages/ScanPage.tsx`:**

    **1. Add signature state:**
    - `const [signatureOpen, setSignatureOpen] = useState(false)`
    - `const [pendingScan, setPendingScan] = useState<{ qrCode: string; photoUrls?: string[] } | null>(null)`
    - `const { user } = useAuthContext()` (import from context)

    **2. Modify single scan flow for delivery action:**
    - In `handleScan` (non-batch mode), when `selectedAction === "delivered"`:
      - Store the scan data in `pendingScan` (qrCode + any photoUrls from pending uploads)
      - Open the signature dialog: `setSignatureOpen(true)`
      - Do NOT fire the mutation yet — wait for signature confirm or skip
    - For non-delivery actions: proceed as before (immediate mutation)

    **3. Handle signature confirm:**
    - `handleSignatureConfirm(blob: Blob)`:
      - Upload PNG via `uploadSignaturePng(blob, "pending", "pending")` → get `signatureUrl`
      - Fire `processMutation.mutate({ qrCode: pendingScan.qrCode, action: "delivered", signatureUrl, photoUrls: pendingScan.photoUrls })`
      - Close dialog, clear pendingScan

    **4. Handle signature skip:**
    - `handleSignatureSkip()`:
      - Fire mutation WITHOUT signatureUrl: `processMutation.mutate({ qrCode: pendingScan.qrCode, action: "delivered", photoUrls: pendingScan.photoUrls })`
      - Close dialog, clear pendingScan

    **5. Receiver auto-detect for signature prompt messaging:**
    - After a successful delivery scan result (from `onSuccess`), the scan already went through with or without signature
    - The signature dialog title can say "Signature for {receiverName}" if the current user is the receiver
    - After scan succeeds with action `delivered`, fetch the shipment's receiver data from the scan result (extend `ScanResult` to include `receiverUid` and `receiverEmail` from the API response)
    - Call `isReceiver(currentUser, { uid: result.receiverUid, email: result.receiverEmail })` to check if the current user is the receiver
    - If receiver match: auto-open signature dialog with title "Sign to confirm delivery"
    - If not receiver: still show a "Capture Signature" button below the success card so the driver can manually trigger it if the receiver is present

    **6. Batch mode signature handling:**
    - For batch mode with delivery action: do NOT show per-item signature dialogs (impractical for batch of 50)
    - Instead, show a single optional signature prompt BEFORE "Confirm All" if the batch action is "delivered"
    - The same signatureUrl gets applied to all scans in the batch (all delivered at once = one signature event)

    **7. Render SignatureDialog:**
    ```tsx
    <SignatureDialog
      open={signatureOpen}
      onConfirm={handleSignatureConfirm}
      onSkip={handleSignatureSkip}
      onClose={() => { setSignatureOpen(false); setPendingScan(null); }}
    />
    ```
