    - `const [pendingPhotos, setPendingPhotos] = useState<File[]>([])`
    - Generate unique IDs for batch items using `crypto.randomUUID()`

    **2. Add batch mode toggle:**
    - Render a toggle switch below the ActionSelector: label "Batch Mode" with a styled checkbox/switch
    - When toggled off, clear `batchQueue` and `batchResults`

    **3. Modify handleScan behavior based on batchMode:**
    - When `batchMode === false` (current behavior): process single scan immediately via `scan.process`, include `photoUrls` if pending photos exist (upload photos first via `uploadScanPhoto`, collect URLs, then mutate)
    - When `batchMode === true`: add to `batchQueue` as a `BatchItem` with the current `selectedAction` and any pending photo URLs. Clear `pendingPhotos` after adding. Do NOT call mutation yet.

    **4. Add batch mutation:**
    - `const batchMutation = trpc.scan.processBatch.useMutation({ ... })`
    - `onConfirmAll` handler: call `batchMutation.mutate({ scans: batchQueue.map(item => ({ qrCode: item.qrCode, action: item.action as ScanAction, photoUrls: item.photoUrls })) })`
    - On success: set `batchResults` from response; play success beep; add successful items to `scannedItems`
    - On error: play error buzz; show toast

    **5. Add batch queue remove handler:**
    - `handleRemove(id: string)`: filter item out of `batchQueue`, clear `batchResults`

    **6. Render new components:**
    - `<PhotoCapture onPhotosChange={setPendingPhotos} disabled={processMutation.isPending || batchMutation.isPending} />` — placed between the ScanInput and the camera button
    - When `batchMode`: show `<BatchScanQueue items={batchQueue} onRemove={handleRemove} onConfirmAll={handleConfirmAll} results={batchResults} isPending={batchMutation.isPending} />` instead of the session scanned list
    - When not `batchMode`: show existing `<ScannedPiecesList>` as before

    **7. Photo upload for single scan mode:**
    - Before calling `processMutation.mutate`, if `pendingPhotos.length > 0`:
      - Show a loading state
      - Upload each file via `uploadScanPhoto(file, "pending", "pending")` (shipmentId/pieceId not known yet — use a temp path or "unassigned" prefix; the URL is what matters since the API stores it on the piece)
      - Collect download URLs into `photoUrls` array
      - Include `photoUrls` in the mutation input
      - Clear `pendingPhotos` after successful mutation
    - Note: For batch mode, photos must be uploaded before adding to queue (not at confirm time) since the queue stores URL strings

    **IMPORTANT:** Batch mode uses the single global action from `ActionSelector` for all queued scans (per discretion — simplest UX). Each queued item inherits `selectedAction` at the time of scan.
