    **useShipmentsSubscription hook** (`apps/web/src/hooks/useShipmentsSubscription.ts`):

    Create a custom React hook that subscribes to the `shipments` Firestore collection using `onSnapshot` (per D-04).

    Query strategy (per D-05, avoiding `not-in` + inequality compound query limitation):
    - `collection(firestore, "shipments")`
    - `where("createdAt", ">", Timestamp.fromDate(thirtyDaysAgo))` — 30-day window
    - `orderBy("createdAt", "desc")`
    - Client-side filter: exclude `picked_up` and `cancelled` statuses unless user explicitly requests them via a `showCompleted` parameter (default `false`)

    The existing composite index `{status ASC, createdAt DESC}` covers this query pattern.

    Hook signature:
    ```typescript
    interface UseShipmentsSubscriptionOptions {
      showCompleted?: boolean; // include picked_up/cancelled, default false
    }
    interface UseShipmentsSubscriptionResult {
      shipments: ShipmentWithId[];
      loading: boolean;
      error: Error | null;
    }
    export type ShipmentWithId = Shipment & { id: string };
    export function useShipmentsSubscription(options?: UseShipmentsSubscriptionOptions): UseShipmentsSubscriptionResult;
    ```

    Implementation:
    - Use `useState` for `shipments`, `loading` (init `true`), `error`
    - Use `useEffect` with `onSnapshot(query, onNext, onError)`:
      - `onNext`: map `snapshot.docs` to `ShipmentWithId[]` (spread `doc.data()` + `id: doc.id`), apply client-side status filter if `showCompleted` is false (exclude `picked_up`, `cancelled`), set `loading = false`
      - `onError`: set `error`, set `loading = false`
    - Return unsubscribe from useEffect cleanup
    - Compute `thirtyDaysAgo` as `new Date(Date.now() - 30 * 24 * 60 * 60 * 1000)` inside the effect
    - Import `collection, query, where, orderBy, onSnapshot, Timestamp` from `firebase/firestore`
    - Import `firestore` from `../firebase`
    - Import `Shipment` from `@material-tracking/shared`

    **Exception classification** (`apps/web/src/utils/exceptions.ts`):

    Create pure functions for classifying shipment exceptions (per D-07).

    ```typescript
    export const ExceptionType = {
      STALLED: "stalled",
      OVERDUE: "overdue",
      AGED: "aged",
    } as const;
    export type ExceptionType = (typeof ExceptionType)[keyof typeof ExceptionType];

    export interface ShipmentException {
      type: ExceptionType;
      shipmentId: string;
    }

    export function classifyExceptions(shipment: ShipmentWithId, now?: Date): ExceptionType[];
    export function classifyAllExceptions(shipments: ShipmentWithId[], now?: Date): Map<string, ExceptionType[]>;
    ```

    Exception rules:
    - **Stalled** (D-07): `updatedAt` is 4+ hours ago AND status is NOT `delivered`, `picked_up`, or `cancelled`. Use `now.getTime() - updatedAt.toDate().getTime() >= 4 * 60 * 60 * 1000`.
    - **Overdue** (D-07): `createdAt` is 24+ hours ago AND status is NOT `delivered`, `picked_up`, or `cancelled`. Use `now.getTime() - createdAt.toDate().getTime() >= 24 * 60 * 60 * 1000`.
    - **Aged** (D-07): status is `delivered` AND `deliveredAt` exists AND `deliveredAt` is 24+ hours ago AND status is NOT `picked_up`. Use `now.getTime() - deliveredAt.toDate().getTime() >= 24 * 60 * 60 * 1000`.

    `classifyAllExceptions` iterates all shipments and returns a `Map<shipmentId, ExceptionType[]>`.

    Use the const object enum pattern consistent with `packages/shared/src/enums.ts`.
