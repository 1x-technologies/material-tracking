    - Add text fields using `.text()`:

  <action>
    Create `apps/web/src/lib/zebra/browserPrint.ts` — a TypeScript module that communicates with the Zebra Browser Print local agent via HTTP.

    The Browser Print agent listens on the local machine. The wrapper should try HTTP on port 9100 first (simpler, no cert issues). The module handles three operations:

    **Exported types:**
    ```typescript
    export interface ZebraPrinter {
      uid: string;
      name: string;
      connection: string;
      deviceType: string;
    }

    export interface BrowserPrintError {
      type: "agent_unavailable" | "no_printers" | "send_failed";
      message: string;
      instructions?: string;
    }
    ```

    **Exported functions:**

    **`checkAgent(): Promise<boolean>`**
    - Fetch GET to `http://localhost:9100/available` with a 3-second timeout
    - Return `true` if response is OK, `false` on any error (network, timeout, etc.)
    - Use `AbortController` for timeout

    **`discoverPrinters(): Promise<{ printers: ZebraPrinter[] } | { error: BrowserPrintError }>`**
    - Call `checkAgent()` first; if false, return error with `type: "agent_unavailable"` and `instructions` string: "Zebra Browser Print is not running. Please ensure the Zebra Browser Print application is installed and running on this device. Download it from zebra.com/browserprint."
    - Fetch GET to `http://localhost:9100/available`
    - Parse response (expected format: JSON or newline-separated device info — adapt based on actual agent response)
    - If no printers found, return error with `type: "no_printers"` and message: "No Zebra printers found. Ensure a printer is connected and powered on."
    - Return `{ printers: [...] }` on success

    **`sendZpl(printer: ZebraPrinter, zpl: string): Promise<{ success: true } | { error: BrowserPrintError }>`**
    - POST ZPL string to the printer's write endpoint
    - The Browser Print agent typically accepts POST to `http://localhost:9100/write` with the device UID and ZPL data
    - On success return `{ success: true }`
    - On failure return error with `type: "send_failed"` and message describing the issue

    **Implementation notes:**
    - All fetch calls should have try/catch wrapping for network errors
    - Use `AbortController` with 5-second timeout for send operations
    - Per D-09: the `instructions` field on `agent_unavailable` error provides user-facing guidance
    - Do NOT import any external SDK — this is a pure fetch-based wrapper
    - Export a `BROWSER_PRINT_BASE_URL` constant (`http://localhost:9100`) so it can be overridden in tests
